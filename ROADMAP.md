# PressTalk Roadmap

## 当前状态

PressTalk 已经完成从 Python 工具到原生 Swift 菜单栏应用的切换，现阶段目标是把它打磨成一个稳定、轻量、长期可维护的 macOS 输入工具。

当前已完成：

- 原生 Swift 菜单栏 App
- 右 `Option` 普通转录
- 右 `Command` 智能整理
- 火山引擎豆包 ASR 接入
- DeepSeek 后处理接入
- 设置页、权限引导、连接测试
- 未签名 `.app` 构建与本地固定安装流程
- 品牌名正式确定为 `PressTalk`

---

## Phase 1 — 稳定性与产品收口

### 1.1 固定安装与授权流程 ✅

推荐使用路径已经明确：

1. 用 Xcode 构建 `PressTalk.app`
2. 放入 `/Applications`
3. 再去授予辅助功能、输入监控、麦克风权限

这样授权路径固定，后续使用最稳定。

### 1.2 权限状态刷新优化 ✅

设置页会在以下时机刷新权限状态：

- 打开设置页
- App 回到前台
- 设置窗口重新获得焦点
- 点击“前往设置”后进入短时轮询

如果系统里已经授权但界面仍未更新，PressTalk 会提示用户重启 App。

### 1.3 品牌基础 ✅

- 产品名统一为 `PressTalk`
- 菜单栏、设置页、README、路线图全部同步
- 新图标与原生项目资源统一

---

## Phase 2 — 输入体验增强

### 2.1 自定义触发键

当前触发键仍固定为右 `Option` / 右 `Command`。下一步考虑支持用户自定义触发键或模式映射。

### 2.2 历史记录

增加本地历史记录，支持快速回看和再次复制最近的转录内容。

### 2.3 更好的错误反馈

把目前主要依赖控制台输出的错误提示进一步前移到界面中，例如：

- API Key 缺失
- 网络不可用
- 识别结果为空
- 权限不完整

---

## Phase 3 — 识别质量与延迟

### 3.1 连接预热 ✅

程序启动后预热 WebSocket 建连，已减少首次转录的等待时间。

### 3.2 自定义词汇表

研究火山引擎热词能力，用于提升人名、产品名、代码词汇的识别准确率。

### 3.3 更低延迟的实时体验

长期目标是探索流式 ASR 与更快的逐步注入体验，但这会显著增加状态管理和输入逻辑复杂度，优先级低于稳定性。

---

## Phase 4 — 分发与系统集成

### 4.1 正式签名与公证

当前默认是未签名构建。后续如果要面向更广泛用户分发，需要补上：

- 代码签名
- Notarization
- 更顺滑的首次启动体验

### 4.2 开机自启

通过 `launchd` 或现代登录项机制实现开机自动启动。

### 4.3 自动更新

当本地分发流程稳定后，再考虑增量更新或 Sparkle 一类方案。
